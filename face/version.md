## 패치 버전 히스토리

### v0.1 — HOG 픽셀 특징 (36%)


처음엔 이미지 픽셀을 통째로 넣어봤다.

HOG라는 방식으로 128×128 이미지에서 숫자 1764개를 뽑아서 RandomForest로 학습했는데 결과가 **36%** 나왔다.

5개 중에 랜덤으로 찍어도 20%가 나오니까 사실상 별로 못 맞추는 결과다.

픽셀은 조명이 밝은지 어두운지, 배경이 뭔지 같은 것도 다 학습해버려서 얼굴 모양 자체를 제대로 못 배웠던 것이다.

```
특징 차원 : 1764 (HOG)
5-Fold CV : 39.75% ± 3.39%
테스트    : 36.0%
```

<br><br><br>  

### v0.2 — MediaPipe 랜드마크 비율 (44%)

MediaPipe FaceLandmarker로 얼굴에 점 478개를 찍고, 그 점들 사이 거리로 비율을 계산했다.

"얼굴 길이 / 광대 너비"처럼 7개 숫자를 뽑아서 학습했더니 **44%** 로 올랐다.

픽셀보다 얼굴 구조 자체를 반영하는 수치니까 방향이 맞다는 신호였다.

```
특징 차원        : 7 (R1/R2/R3/upper_r/mid_r/lower_r/golden_flag)
5-Fold CV        : 45.31% ± 6.58%
테스트           : 44.0%
특징 중요도 1위  : R3 광대 돌출도 (0.2049)
```

<br><br><br>  

### v0.3 — 얼굴 크롭 2-pass (46%)

원본 이미지 전체에서 점을 찍으면 배경이 끼어들어서 정밀도가 낮다.

그래서 **2-pass** 방식을 도입했다. 

먼저 원본에서 얼굴 위치를 파악하고 얼굴 부분만 잘라낸 뒤 거기서 다시 점을 찍는다.

배경 없이 얼굴만 보니까 수치가 더 일관되게 나왔다.

```
5-Fold CV : 43.44% ± 4.57%
테스트    : 46.0%
```

<br><br><br>  

### v0.4 — 턱 각도·면적 특징 추가 (56%)

v0.3까지는 비율 숫자만 썼는데 

heart·oblong·oval·round 4개 클래스의 광대 돌출도(R3)가 거의 겹쳐 있어서 구분이 안 됐다.

그래서 턱선의 **모양**을 나타내는 특징 3개를 추가했다.

| 특징 | 의미 | 효과 |
|------|------|------|
| `jaw_angle` | 귀밑→턱각→턱끝 각도 | 각진형(143°) vs 둥근형(150°) 구분 |
| `lower_area_r` | 하악 삼각형 면적 / 얼굴 전체 면적 | 뾰족 턱 vs 넓은 턱 구분 |
| `R4` | 광대 너비 / 턱 너비 | 하트형 vs 각진형 구분 |

jaw_angle 하나만으로 각진형과 나머지를 7° 차이로 구분할 수 있게 됐다.

결과: **46% → 56%** (+10%p). 전체 버전 중 단일 개선 최대폭이다.

```
특징 차원        : 10개
5-Fold CV        : 47.19% ± 2.30%
테스트           : 56.0%
특징 중요도 1위  : jaw_angle (0.1632)
```

<br><br><br>  

### v0.5 — 앞머리 필터 + 앙상블 + Top-2 추론 (55%)

두 가지를 바꿨다.

<br>

**앞머리 필터** — 앞머리가 있으면 헤어라인 스캔이 막혀서 이마 비율(upper_r)이 비정상적으로 낮게 잡힌다. 

`upper_r < 0.15`인 이미지 31장(6.2%)을 학습에서 제외했다.

<br>

**앙상블 모델** — RandomForest 하나 대신 RF + SVM(RBF) + HistGradientBoosting 3가지 모델의 예측 확률을 평균 냈다. 

각 모델이 서로 다른 경계를 학습해서 약점을 보완하는 구조다.

<br>

**Top-2 추론** — 1위만 보여주는 게 아니라 1·2위를 함께 보여주고 사용자가 고르게 했다.

```
학습 이미지 : 469장 (앞머리 31장 제외)
특징 차원   : 12개
5-Fold CV   : 57.33% ± 6.46%
테스트      : 55.32%
```

<br><br><br>  

### v0.6 — 비대칭 필터 추가 (역효과, 55%)

측면으로 찍힌 사진은 얼굴이 비대칭으로 보이니까 필터로 제거하려 했다.

`symmetry_r > 0.18` 조건으로 이미지 80장(16%)을 추가로 제거했는데, 

학습 데이터가 389장으로 줄어서 오히려 CV 정확도가 떨어졌다.

데이터가 적은 상황에서는 공격적인 필터링이 역효과라는 걸 확인했다.

```
학습 이미지 : 389장 (비대칭 80장 추가 제외)
특징 차원   : 14개
5-Fold CV   : 48.35% ± 4.38%
테스트      : 55.13%
반복 평가   : 58.53% (seed 0~19 평균)
```

<br><br><br>  

### v0.7 — 비대칭 필터 제거 + 이중 앙상블 모드 (58%)

v0.6 교훈을 반영해 비대칭 필터를 없앴다. `symmetry_r`은 제거 기준이 아니라 학습 특징으로만 사용했다.

`--stable-model` 플래그를 추가해서 앙상블 하이퍼파라미터를 두 가지 모드로 비교할 수 있게 했다.

```
학습 이미지    : 469장 (비대칭 필터 없음)
기본 모드      : 테스트 54.26%
--stable-model : 테스트 58.51% ← 당시 최고
반복 평가      : 58.67% (seed 0~19)
```

<br><br><br>  

### v0.8 — 기하학 비율 + 크롭 픽셀 PCA 복합 특징 (CV 62%, 테스트 54%)

비율 숫자만으로는 못 잡는 얼굴 윤곽 패턴을 이미지 픽셀이 보완할 수 있다는 아이디어다.

크롭한 얼굴을 64×64 그레이스케일로 저장하고, PCA(50)로 압축한 뒤 기하학 비율 15개와 합쳐서 **65차원** 복합 특징으로 학습했다.

CV 정확도는 62%로 당시 최고였지만, 데이터가 500장밖에 없어서 단일 테스트(54%)와 괴리가 컸다. 

데이터가 부족하면 테스트 분할 운에 따라 결과가 크게 흔들린다는 걸 확인했다.

```
특징 구성   : 기하학 15개 + 크롭 픽셀 → PCA 압축 → 65차원
5-Fold CV   : 62.33% ± 3.09% ← 당시 최고
테스트      : 54.26% (데이터 부족으로 split 운 영향 큼)
```

<br><br><br>  

### v0.9 — 사전 분리 데이터 활용 (65%)

v0.1~v0.8은 500장을 코드 안에서 나눠 썼는데, v0.9부터 미리 분리된 대규모 데이터셋을 그대로 사용했다.

```
기존: dataset/ 500장 → 코드 안에서 80/20 분리
v0.9: training_set/ 4000장 (앞머리 제외 후 3861장)
      testing_set/  1000장 (앞머리 제외 후  960장)
```

<br>

코드 거의 안 바꾸고 데이터만 8배 늘렸는데 **65%** 로 올랐다.

CV 표준편차도 ±0.36%로 전 버전 최소 — 데이터가 많을수록 결과가 안정적이다.

```
5-Fold CV     : 63.53% ± 0.36%
테스트        : 65.10%
특징 중요도 1위: R1 얼굴 길이감 (0.1582)
```

### v1.0 — 3-pass 크롭 표준화 (역효과, 55%)

크롭을 한 번 더 해서 3-pass로 만들고, 이미지를 256×256으로 표준화했다.

의도는 좋았는데 문제가 생겼다. 

256px로 강제 리사이즈하면 가로형·세로형 얼굴이 같은 크기가 되면서 **얼굴 길이/너비 비율(R1)이 왜곡**됐다. 

R1이 핵심 특징인데 그게 망가진 거다.

비율 기반 특징에는 원본 비율을 유지하는 게 더 중요하다는 걸 배웠다.

```
5-Fold CV : 54.54% ± 0.66%
테스트    : 55.23% ← v0.9 대비 -10%p
```

### v1.1 — 17특징 + StackingClassifier (66%)

특징을 12개에서 17개로 늘리고, 앙상블 방식을 VotingClassifier에서 StackingClassifier로 바꿨다.

Voting은 3개 모델 확률을 단순 평균 내지만 

Stacking은 메타 학습기(LogisticRegression)가 각 모델의 신뢰도를 데이터로 배워서 더 적응적으로 결합한다.

새로 추가한 특징 중 `chin_angle`(턱끝 뾰족함)의 클래스 구분력(F-ratio)이 876으로 가장 높았다.

```
특징 차원        : 17개
앙상블           : StackingClassifier (HGB + SVM + RF, meta=LR)
5-Fold CV        : 65.76% ± 1.01%
테스트           : 66.56%
특징 중요도 1위  : chin_angle (F-ratio 876.84)
```

<br><br><br>  

### v1.2 — v0.8 구조 × 대규모 데이터 (69%)

v0.8의 기하학+픽셀 복합 특징 구조를 그대로 가져와서 대규모 데이터(3861장)로 학습했다.

v0.8에서 CV는 62%였는데 테스트가 54%로 낮았던 이유가 500장 소규모 때문이었다. 

8배 더 많은 데이터로 학습하니까 **69%** 로 올랐다.

```
특징 구성  : 기하학 15개 + 크롭 픽셀 → 65차원
5-Fold CV  : 66.93% ± 1.32%
테스트     : 69.06% ← ML 기반 전체 최고
```

### v1.3 — EfficientNet-B0 임베딩 도입 (71%)

지금까지는 픽셀을 직접 PCA로 압축했는데 

여기서는 ImageNet으로 미리 학습된 **EfficientNet-B0**를 특징 추출기로 사용했다.

EfficientNet은 수백만 장 이미지로 학습해서 얼굴 윤곽·형태에 대한 표현이 이미 잘 만들어져 있다. 

분류기 부분만 제거하고 1280차원 벡터를 뽑아서 기하학 15차원과 합쳐 학습했다.

2가지 모델을 비교해서 더 좋은 걸 저장했다.

```
LightGBM 단독              : 68.23%
Stacking (HGB+LGBM+SVM)   : 71.88% ← 채택
총 입력 차원               : 1295 (기하학 15 + 임베딩 1280)
```

### v1.4 — torchvision EfficientNet + CatBoost + 5-base Stacking

v1.3의 timm 라이브러리 의존성을 torchvision으로 교체하고 CatBoost를 추가해서 3가지 모델을 비교했다.

기하학 특징과 임베딩 특징을 각각 StandardScaler로 따로 정규화해서 SVC 성능을 높였다.

데이터 누수 방지를 위해 Scaler는 학습 데이터에만 fit하고 테스트 데이터는 transform만 적용했다.

EfficientNet 임베딩을 제거하고


### v1.7 — EfficientNet-B4 파인튜닝 (82%)

v1.3은 EfficientNet을 특징 추출기로만 썼는데 v1.7부터는 얼굴형 데이터로 **직접 파인튜닝**했다.

파인튜닝이란 미리 학습된 모델을 우리 데이터에 맞게 추가 학습하는 거다. 

특징만 뽑을 때보다 훨씬 잘 맞춘다.

ML 기반 최고(69%)에서 CNN 파인튜닝으로 단번에 **82%** 로 뛰었다.

```
모델    : EfficientNet-B4 (224×224)
총 학습 : 88 epoch (EarlyStopping 발동)
Best    : 82.70% (epoch 73)
```

<br><br><br>  

### v1.8 — 380px + 2-Phase + Mixup (87%)

EfficientNet-B4의 원래 입력 크기는 380×380인데, 그동안 224×224로 줄여서 넣고 있었다.

원래 크기로 맞추고 학습을 두 단계로 나눴다.

- **Phase 1 (epoch 1~10)**: 뒷부분(head)만 학습 → 기존 ImageNet 특징 보존

- **Phase 2 (epoch 11~154)**: 전체 학습 → 얼굴형에 맞게 세밀하게 조정 + Mixup 증강 적용

<br>

**Mixup** — 두 이미지를 반투명하게 겹쳐서 새 학습 데이터를 만드는 기법이다. 

모델이 더 다양한 경우를 학습하게 된다.

```
모델    : EfficientNet-B4 (380×380)
총 학습 : 154 epoch
Best    : 87.40% (epoch 134)
```

<br><br><br>  

### v1.9 — EfficientNetV2-S + CutMix + SWA + TTA (90.9%)

네 가지를 한꺼번에 바꿨다.

- **① EfficientNetV2-S로 교체** — B4보다 최신 모델이고 ImageNet 정확도도 더 높다.

  입력 크기 384×384가 딱 맞는다.

- **② CutMix + Mixup 혼합 증강** — Mixup은 두 이미지를 반투명하게 겹치는 방식. 

  CutMix는 한 이미지 일부를 잘라서 다른 이미지에 붙이는 방식이다. 둘을 50:50으로 랜덤하게 사용했다.

- **③ SWA (가중치 평균)** — 학습이 수렴하면 그 이후 20번의 모델 가중치를 전부 평균 냈다. 

  한 번 잘한 모델보다 평균 모델이 더 안정적으로 잘 맞춘다.

- **④ TTA (5번 예측 평균)** — 이미지 한 장을 5가지 방식으로 변형해서 각각 예측하고 평균 냈다.

```
3-Phase 학습:
  Phase 1 (1~10 epoch)    : head만 학습
  Phase 2 (11~100 epoch)  : 전체 학습 + EarlyStopping(patience=25)
  Phase 3 (101~120 epoch) : SWA 수집 → swa_model.pth 저장

Best 모델 단독   : 89.80%
Best 모델 + TTA  : 90.30%
SWA 모델 + TTA   : 90.90% ← 최종
```

## 남성 전이학습 — EfficientNetV2-S 남성 파인튜닝

여성 SWA 모델(`female/model/swa_model.pth`)을 초기 가중치로 사용해서 남성 얼굴형 데이터를 파인튜닝했다.

여성 모델을 처음부터 다시 학습하지 않고, 이미 얼굴 구조를 배운 여성 가중치에서 출발하는 방식이다.

<br>

### 전이학습 전략

EfficientNetV2-S의 레이어를 두 그룹으로 나눠서 다르게 처리했다.

| 레이어 | 처리 | 이유 |
|--------|------|------|
| `features[0~4]` | **동결** | 에지·질감·기본 얼굴 구조 — 성별 공통 |
| `features[5~7]` + `classifier` | **파인튜닝** | 고수준 윤곽·비율 — 성별 차이 반영 |

하위 레이어는 성별과 무관하게 공통적인 얼굴 특징(윤곽선, 피부 질감)을 담고 있기 때문에 동결해서 기존 지식을 보존했다.

상위 레이어는 "이마가 좁은지 넓은지", "턱이 둥근지 각진지" 같은 고수준 비율을 판단하는데, 남성 얼굴의 특성을 반영하도록 파인튜닝했다.

<br>

### 남성 데이터셋 클래스 매핑

남성 데이터셋 폴더명이 여성 클래스명과 달라서 매핑이 필요하다.

| 남성 폴더명 | 매핑 클래스 |
|---|---|
| rectangular | Oblong |
| ovale | Oval |
| round | Round |
| square | Square |

Heart 클래스는 남성 데이터가 없어서 여성 모델 가중치를 그대로 유지했다.

남성 모델에서 Heart가 예측되면 참고치로만 활용한다.

<br>

### 하이퍼파라미터

| 항목 | 값 |
|------|-----|
| 백본 | EfficientNetV2-S (384×384) |
| 배치 크기 | 8 (Gradient Accum × 2 = 실효 16) |
| 파인튜닝 에폭 | 50 (기본) |
| SWA 에폭 | 마지막 10 epoch |
| Backbone LR | 1e-5 |
| Head LR | 1e-4 |
| SWA LR | 1e-6 |
| EarlyStopping patience | 10 |
| 스케줄러 | CosineAnnealingWarmRestarts (T_0=20) |

<br>

### 배경 제거

rembg AI 모델로 배경만 흰색으로 교체하고 머리카락·얼굴은 보존한다.

rembg가 없는 환경에서는 OpenCV GrabCut으로 폴백한다.

배경 노이즈를 제거하면 얼굴 윤곽 학습에 집중할 수 있어서 남성처럼 데이터가 적은 경우에 효과가 크다.

<br>

### 증강

| 증강 | 설명 |
|------|------|
| RandomCrop | 384px로 랜덤 크롭 |
| RandomHorizontalFlip | 좌우 반전 |
| RandomRotation | ±10° 회전 |
| ColorJitter | 밝기·대비·채도·색조 랜덤 변형 |
| RandomErasing | 이미지 일부 마스킹 (과적합 방지) |

CutMix/Mixup은 여성 학습에만 적용하고 남성 파인튜닝에서는 제외했다.

남성 데이터가 적기 때문에 레이블이 섞이는 증강보다 공간·색상 변형 위주로 구성했다.
